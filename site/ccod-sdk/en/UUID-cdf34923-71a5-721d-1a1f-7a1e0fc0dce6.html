CashComplete™ On-Device operator service menu

CashComplete™ On-Device usually has a service menu where the operator can log in to perform service tasks, such as resetting devices and resolving errors. The appearance of the service menu differs between different CashComplete™ On-Device applications. In the RCS, it is a separate Status window. You use the keyboard alt+TAB keys to open the window. You have to log in with a user with SERVICE_MODE permissions. It is only possible to log in when the system is in the standard idle/login state, that is, waiting for the user to identify.

If you have implemented your own login procedure as a boot flow, and you never return to the standard idle flow, it is important to make it possible to log in to the service menu. If you don’t, the operator cannot clear errors.

When someone logs in to the service menu, it is important to close all hardware devices. An out-of-use screen should be shown. It is not possible to log in to the service menu when another user is logged in. This is why it is normally only allowed when the system is idle state.

For an SDK plug-in, this can be solved by registering an operator login handler by calling setOperatorLoginHandler(IOJOperatorLoginHandler) in IOJUserManager. If you do not do this, it’s not possible to log in to the service menu while inside your flow. If you have registered an operator login handler, CashComplete™ On-Device calls the handler as soon as someone tries to login to the service menu. You should decide whether or not to allow someone to log in to the service menu at this time. If you want to allow it, you should return true. Then you also must make sure that you close all devices and return from your flow as soon as possible. The service menu does not appear until your flow has exited. If you take to long to exit, the user experience will be bad.

Your flow is called again as soon as the operator logs out, but with a new instance of the flow session and a new CashComplete™ On-Device session is created.
