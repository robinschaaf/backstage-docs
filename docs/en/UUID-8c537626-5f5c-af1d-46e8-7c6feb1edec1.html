Revision history

This is the CashComplete™ On-Device Software Development Kit revision history:

Revision

Revision remark

3.0.0

  • Updated the information on icons for services, SDK installation, and configuration handling.

2.2.0

  • Added Purchase Manager that enables payments (both cash and cashless) with CashComplete™ On-Device through the SDK.

For more information, see Purchases, payments, and kiosks.

2.1.0

  • Possible to use CashComplete™ On-Device translation system

  • Low level serial port implementation through the SDK

  • Possible to use external libs

2.0.0

  • See SDK 2.0.0 for a description of all the news in version 2.0.0 of the SDK

  • CashComplete™ On-Device 3.0.0 introduces icons for services.

1.4.

Added move notes to transport box

1.4.0

  • Support for all variants of RCS 400

  • Simplified dispense handling. Deprecated the concept of “dispense destination” for coins.

  • New door handling that supports more machine types

  • Support for more variants of I/O boards (e.g. RCS 400 and CDS9)

1.3.9

Added getExpectedAmount() on IOJUser with option to get remaining or originally set amount(s)

1.3.8

Added setExpectedAmount() on IOJUser to allow setting multiple expected amounts by Currency

1.3.7

Added getExpectedAmount() and setExpectedAmount() on IOJUser to allow to set expected amounts for the user during the standard CCoD deposit flow

1.3.6

Added IOJKeyListener2 for support for custom screens

1.3.5

UNLIMITED IOJDispenseDestination added

1.3.2

IOJTransaction2 added

1.3.1

  • Transaction with additionalMoney always created in doDeposit

  • CAM Error 138 handled correctly

1.3.0

Added possibility to hide CashComplete™ On-Device windows

1.2.0

  • Added support for work unit

  • Added possibility of having a margin to the left of barcodes printed with OJReceipt.addBarcode()

  • Added possibility to cancel IOJDepositManager.beginTransaction() and IOJDispenseManager.doDispense() when they are waiting for till to be inserted/removed.

  • Support for rear access (IOJOperatorLoginHandler2 and automatic printer selection)

1.1.1

  • Added new transaction types (IOJTransaction.OJTransactionType)Since release

  • Since release 1.10.0 of CashComplete™ On-Device RCS Software the restriction that JRE 1.6.0_u11 or newer was not allowed has been removed (you can now run with any JRE 1.6)

1.1.0

  • IOJCashDevice.getDepositHandler() can be called from monitoring flows and from isVisible() and isEnabled() in the service flow (but no methods in the deposit handler can be called, this can only be used to check the deposit type).

  • Added setLocale() and resetLocale() to IOJSystemManager

  • Added the interface IOJCashBoxUpdater that allows updates of cash box content (empty and refill cassettes)

  • Deprecated printCustomReceipt() in IOJPrinter and replaced it with printReceipt(). Added classes OJAbstractReceipt and OJReceipt.

  • Added checks to make sure that no flow, including the monitoring flow, is started until all devices are running.

  • “Extra Data” added to drop safe transactions as well

  • Added possibility to present decimals in inputNumber (IOJUIManager)

  • Added method getStandardReceiptProperties() to IOJPrinter.

1.0.0

First official release

0.2.1

Minor updates after review

0.2.0

First Beta version
