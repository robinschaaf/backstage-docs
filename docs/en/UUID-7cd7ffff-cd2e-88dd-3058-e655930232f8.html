SDK managers

When the run() method of the flow is called, it receives a reference to an interface called IOJFlowSession. It is through this interface that the plug-in receives access to the inner parts of CashComplete™ On-Device.

There are some things worth noticing about the flow session:

  • Every time a flow is entered, you get a new instance of the flow session.

  • The same is true for all objects retrieved from the flow session. Even if an object represents the same physical device, you get a new instance when the flow is called again.

  • No method in the flow session, or in any object retrieved from the flow session, maybe called when the flow has exited. This could happen if you create a new thread in your flow.

    If you do call a method after the flow has exited, an exception is immediately thrown.

Through the flow session, you can retrieve manager objects that are responsible for different aspects of CashComplete™ On-Device. The interaction with CashComplete™ On-Device takes place through these managers, or the objects retrieved from the managers.
